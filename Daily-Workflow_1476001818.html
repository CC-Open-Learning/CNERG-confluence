CNERG F24 : Daily Workflow

Created by ASHISH GYAWALI, last modified on Nov 20, 2024

1. Start from the main branch [or the base branch (cnerg-f24-digitaltwin)]

Command to know working branch: git rev-parse --abbrev-ref HEAD

Switch to the main branch

git checkout main

Fetch the latest changes from the remote repository

git fetch origin

Pull the latest changes into your local main branch

git pull origin main

2. Create or Switch to Your Feature Branch

Create and switch to a new feature branch from main (replace "feature-branch" with your branch name)

git checkout -b feature-branch

OR if you already have a feature branch, switch to it

git checkout feature-branch

3. Make Changes and Commit

#Start Developing here

Add your changes to the staging area

git add .

Commit your changes with a descriptive message

git commit -m "Implement new feature"

4. Sync Your Feature Branch with main (using merge)

Fetch the latest changes from the remote

git fetch origin

Merge the latest changes from the main branch into your feature branch

git merge origin/main

If there are conflicts, resolve them, then add the resolved files

git add <file-with-conflict>

Complete the merge

git commit

5. Test Your Feature

Run your tests to make sure everything works after the merge

6. Push Your Changes to the Remote Repository

Push your feature branch to the remote repository

git push origin feature-branch

7. Create a Pull Request (PR)

Go to your Git hosting platform (GitHub, GitLab, Bitbucket), and create a Pull Request (PR) to merge your feature branch into the main branch.

8. Merge the Pull Request

Once the PR is reviewed and approved, merge it into the main branch via the Git hosting platform's web interface.

OR if merging manually, follow these steps:

Switch to the main branch

git checkout main

Pull the latest changes from the remote main branch

git pull origin main

Merge your feature branch into the main branch

git merge feature-branch

Push the merged changes to the remote repository

git push origin main

9. Clean Up Your Feature Branch

Delete the local feature branch

git branch -d feature-branch

Delete the remote feature branch

git push origin --delete feature-branch
