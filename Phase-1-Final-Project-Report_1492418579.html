CNERG F24 : Phase 1 Final Project Report

Created by Eunie, last modified on Nov 27, 2024

1. Executive Summary

Objective:

The Smart Campus Digital Twin project collects real-time foot traffic data using motion sensors across the campus. The data is analyzed to provide insights on infrastructure usage, congestion zones, and movement patterns and visualized through 3D simulations for better decision-making.

Key Outcomes:

  • Successful foot traffic data collection using motion sensors.

  • Insights derived from data analysis to identify traffic patterns and optimize resource allocation.

  • 3D simulation and visualization of traffic patterns using Unity.

2. Methodology

Project Phases:

  1. Planning:

    • Proposal creation and requirement gathering.

    • Identifying sensor placement and data collection requirements.

  2. Implementation:

    • Motion sensors are installed in key locations on campus. (3G26)

    • Database setup using InfluxDB for efficient data storage and retrieval.

  3. Visualization:

    • Integrated data visualization into Unity.

    • Developed 3D simulations to represent foot traffic patterns and movements.

  4. Analysis:

    • Traffic data was analyzed to generate insights on peak usage times and congestion zones.

Tools and Technologies:

  • Simulation Tools: Unity (including Tilt Five SDK, XChart, and NPC tools for simulation).

  • Database: InfluxDB for storing and managing foot traffic data.

  • Motion Sensors: Motion detectors(HX-HE3) are installed in high-traffic areas.

  • Data Analysis Tools: Python for statistical analysis and visualization frameworks.
